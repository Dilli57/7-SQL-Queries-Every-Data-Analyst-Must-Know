01 JOIN

Combine rows from two or more tables based on a related column. The foundation of relational data analysis.

SELECT o.order_id, c.name, o.total_amount FROM orders o INNER JOIN customers c ON o.customer_id = c.id WHERE o.status = 'completed';
INNER LEFT RIGHT FULL OUTER
02 GROUP BY

Aggregate rows sharing common values. Pairs with COUNT, SUM, AVG, MAX, MIN to summarize data.

SELECT region, COUNT(*) AS total_orders, SUM(revenue) AS total_revenue, AVG(revenue) AS avg_order FROM sales GROUP BY region HAVING SUM(revenue) > 10000 ORDER BY total_revenue DESC;
Aggregation Summarization HAVING
03 WINDOW FUNCTIONS

Perform calculations across rows related to the current row — without collapsing them like GROUP BY does.

SELECT employee, dept, salary, AVG(salary) OVER ( PARTITION BY dept ) AS dept_avg, salary - AVG(salary) OVER ( PARTITION BY dept ) AS diff_from_avg FROM employees;
PARTITION BY ORDER BY ROWS/RANGE
04 CASE

SQL's conditional logic. Build custom categories, flag data, or pivot rows into columns on the fly.

SELECT customer_id, total_spend, CASE WHEN total_spend >= 5000 THEN 'Platinum' WHEN total_spend >= 1000 THEN 'Gold' WHEN total_spend >= 250 THEN 'Silver' ELSE 'Bronze' END AS tier FROM customers;
Segmentation Binning Pivoting
05 CTE

Common Table Expressions create named temporary result sets, making complex queries modular and readable.

WITH monthly_revenue AS ( SELECT DATE_TRUNC('month', sale_date) AS month, SUM(amount) AS revenue FROM sales GROUP BY 1 ) SELECT month, revenue, revenue - LAG(revenue) OVER (ORDER BY month) AS mom_change FROM monthly_revenue;
Readability Reuse Recursive
06 SUBQUERY

A query nested inside another. Use in SELECT, FROM, or WHERE to filter or compute against derived results.

SELECT product_name, price FROM products WHERE price > ( -- scalar subquery: single value SELECT AVG(price) FROM products ) AND category_id IN ( SELECT id FROM categories WHERE active = true );
Scalar Correlated IN / EXISTS
07 RANK

Assign rankings to rows within partitions. RANK skips ties, DENSE_RANK doesn't, ROW_NUMBER is always unique.

SELECT salesperson, region, revenue, RANK() OVER ( PARTITION BY region ORDER BY revenue DESC ) AS rank, DENSE_RANK() OVER ( PARTITION BY region ORDER BY revenue DESC ) AS dense_rank FROM sales_performance;
RANK() DENSE_RANK() ROW_NUMBER()
